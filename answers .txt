
По масштабируемому процессингу - как организовать?

Для масштабируемого процессинга можно использовать распределенную очередь задач, такую как Celery, вместе с брокером сообщений, например Redis или RabbitMQ. Это позволит распределять задачи по нескольким рабочим процессам, что увеличит производительность и устойчивость системы. Также стоит рассмотреть возможность использования Kubernetes для оркестрации контейнеров, чтобы автоматизировать развертывание, масштабирование и управление контейнеризированными приложениями.

Как сделать надежное свое S3 хранилище?

Для создания надежного S3 хранилища можно рассмотреть использование MinIO — высокопроизводительной распределенной системы хранения объектов, совместимой с S3 API. MinIO может быть развернут локально или в облаке и обеспечивает высокую доступность данных за счет репликации и распределения данных по нескольким узлам. Также важно настроить резервное копирование и мониторинг состояния системы для своевременного обнаружения и устранения потенциальных проблем.

Если будет желание, сделать минимальную веб версию для тестов

Для создания минимальной веб-версии можно использовать Django вместе с фронтенд-фреймворком, таким как React или Vue.js. Создайте простую форму для загрузки изображений и отображения списка загруженных изображений с их статусами. Используйте WebSocket для оповещения клиента о завершении обработки изображений в реальном времени.
